## Base URL

```
http://localhost:3000
```

## Projects

### Get All Projects

<Badge text="GET" variant="success" />

```
/api/v1/projects
```

Retrieves a paginated list of projects for an organization.

**Query Parameters:**
- `orgId` (required): Organization ID
- `page` (required): Page number (starts from 1)
- `pageSize` (required): Number of items per page
- `textSearch` (optional): Search projects by name

**Example Request:**
```bash
curl -X GET "http://localhost:3000/api/v1/projects?orgId=1&page=1&pageSize=10&textSearch=API" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

**Example Response:**
```json
{
  "data": {
    "projects": [
      {
        "projectId": 1,
        "projectName": "API Testing Project",
        "orgId": 1,
        "description": "Test suite for REST APIs",
        "createdBy": "john.doe@example.com",
        "createdOn": "2024-01-15T10:30:00Z",
        "isArchived": false
      }
    ],
    "totalCount": 25,
    "page": 1,
    "pageSize": 10
  },
  "status": 200
}
```

### Create Project

<Badge text="POST" variant="note" />

```
/api/v1/project/create
```

Creates a new project.

**Request Body:**
```json
{
  "projectName": "My New Project",
  "description": "Project description",
  "orgId": 1

```

**Example Request:**
```bash
curl -X POST http://localhost:3000/api/v1/project/create \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "projectName": "Mobile App Testing",
    "description": "Test cases for iOS and Android apps",
    "orgId": 1
```

**Example Response:**
```json
{
  "data": {
    "projectId": 42,
    "projectName": "Mobile App Testing",
    "orgId": 1,
    "description": "Test cases for iOS and Android apps",
    "createdBy": "user@example.com",
    "createdOn": "2024-01-15T10:30:00Z",

  },
  "status": 201
}
```

## Tests

### Get Tests

<Badge text="GET" variant="success" />

```
/api/v1/project/tests
```

Retrieves tests for a project with filtering and pagination.

**Query Parameters:**
- `projectId` (required): Project ID
- `page` (required): Page number
- `pageSize` (required): Items per page
- `sectionIds` (optional): Comma-separated section IDs
- `priorityIds` (optional): Comma-separated priority IDs
- `labelIds` (optional): Comma-separated label IDs
- `automationStatusIds` (optional): Comma-separated automation status IDs
- `textSearch` (optional): Search in title/description
- `testCoveredBy` (optional): Filter by coverage type
- `platformIds` (optional): Comma-separated platform IDs
- `squadIds` (optional): Comma-separated squad IDs

**Example Request:**
```bash
curl -X GET "http://localhost:3000/api/v1/project/tests?projectId=1&page=1&pageSize=50&priorityIds=1,2" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

**Example Response:**
```json
{
  "data": {
    "tests": [
      {
        "testId": 101,
        "title": "User login with valid credentials",
        "description": "Verify user can login with correct username and password",
        "projectId": 1,
        "sectionId": 5,
        "priorityId": 1,
        "automationStatusId": 2,
        "createdBy": "john@example.com",
        "createdOn": "2024-01-10T09:00:00Z",
        "updatedBy": "jane@example.com",
        "updatedOn": "2024-01-12T14:30:00Z"
      }
    ],
    "totalCount": 150,
    "page": 1,
    "pageSize": 50
  },
  "status": 200
}
```

## Runs

### Get Runs

<Badge text="GET" variant="success" />

```
/api/v1/project/runs
```

Retrieves test runs for a project.

**Query Parameters:**
- `projectId` (required): Project ID
- `page` (required): Page number
- `pageSize` (required): Items per page
- `isLocked` (optional): Filter by locked status (true/false)
- `textSearch` (optional): Search by run name

**Example Request:**
```bash
curl -X GET "http://localhost:3000/api/v1/project/runs?projectId=1&page=1&pageSize=20&isLocked=false" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

**Example Response:**
```json
{
  "data": {
    "runs": [
      {
        "runId": 15,
        "runName": "Sprint 23 Regression",
        "projectId": 1,
        "createdBy": "qa@example.com",
        "createdOn": "2024-01-15T08:00:00Z",
        "isLocked": false,
        "lockedBy": null,
        "lockedOn": null
      }
    ],
    "totalCount": 45,
    "page": 1,
    "pageSize": 20
  },
  "status": 200
}
```

### Get Run Details

<Badge text="GET" variant="success" />

```
/api/v1/run
```

Retrieves detailed information about a specific run, including all tests and their statuses.

**Query Parameters:**
- `runId` (required): Run ID
- `page` (required): Page number
- `pageSize` (required): Items per page
- `textSearch` (optional): Search tests in run
- `sectionIds` (optional): Filter by sections
- `priorityIds` (optional): Filter by priorities
- `status` (optional): Filter by test status
- `testedBy` (optional): Filter by tester
- `squadIds` (optional): Filter by squads

**Example Request:**
```bash
curl -X GET "http://localhost:3000/api/v1/run?runId=15&page=1&pageSize=100" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## Bulk Operations

### Upload Tests (CSV)

<Badge text="POST" variant="note" />

```
/api/v1/tests/upload
```

Bulk upload tests from CSV file.

**Request:**
- Content-Type: `multipart/form-data`
- Field name: `file`
- File format: CSV

**CSV Format:**
```csv
title,description,sectionId,priorityId,automationStatusId,steps,expectedResult
"Test 1","Description 1",5,2,1,"Step 1\nStep 2","Expected result"
"Test 2","Description 2",5,1,2,"Step 1\nStep 2","Expected result"
```

**Example:**
```bash
curl -X POST http://localhost:3000/api/v1/tests/upload \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "file=@tests.csv" \
  -F "projectId=1"
```

## Error Handling

All endpoints return standard error responses:

### 400 Bad Request
```json
{
  "error": "Invalid param projectId",
  "status": 400
}
```

### 401 Unauthorized
```json
{
  "error": "Unauthorized - Please login",
  "status": 401
}
```

### 403 Forbidden
```json
{
  "error": "User does not have access",
  "status": 403
}
```

### 404 Not Found
```json
{
  "error": "Project not found",
  "status": 404
}
```

### 500 Internal Server Error
```json
{
  "error": "Internal server error",
  "status": 500
}
```
